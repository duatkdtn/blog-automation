[ 3. GitHub Actions 인증 방식 전면 개선 ]
문제: token.pickle이 만료/충돌되어 GitHub Actions 발행 실패 반복
원인: get_google_credentials()가 token.pickle만 사용, GOOGLE_TOKEN 환경변수 무시

해결:
- get_google_credentials() 수정
  → GOOGLE_TOKEN 환경변수 있으면 refresh_token으로 직접 인증
  → 없으면 기존 token.pickle 방식 유지 (로컬 사용 시)
- auto_publish.py restore_token() 제거
  → 기존: base64로 token.pickle 복원 시도 → 실패
  → 변경: get_google_credentials()가 직접 처리하므로 불필요
- auto_publish.py setup_env()에 GOOGLE_CLIENT_ID/SECRET 추가
- auto_publish.yml에 환경변수 추가:
  GOOGLE_CLIENT_ID / GOOGLE_CLIENT_SECRET

GitHub Secrets 추가 목록:
  GOOGLE_TOKEN       ← refresh_token 값 (get_refresh_token.py로 추출)
  GOOGLE_CLIENT_ID   ← Google OAuth 클라이언트 ID
  GOOGLE_CLIENT_SECRET ← Google OAuth 클라이언트 시크릿

[ 4. 유틸 스크립트 생성 ]
